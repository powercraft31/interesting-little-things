# v6.9 Production Upgrade Checklist — Shared-Host Safe

Scope: promote Solfacil only. Do not disturb unrelated host services.

## Preconditions
- Target repo commit: `2afdf57`
- Use compose override: `docker-compose.production.yml`
- Use DB reset path: `backend/scripts/bootstrap-production.sh`
- Do **not** rely on `/docker-entrypoint-initdb.d` demo seeds for production reset.

## Solfacil-only objects allowed to change
- `/opt/solfacil` repo checkout
- containers:
  - `solfacil-db`
  - `solfacil-bff`
  - `solfacil-m1`
- Docker volume:
  - `solfacil_solfacil_pgdata`

## Shared-host objects forbidden by default
- `nginx-web`
- host-wide `redis`
- unrelated containers/services/listeners

## Execution order
1. Backup current DB and record current commit SHA.
2. Update `/opt/solfacil` to `2afdf57`.
3. Start only Solfacil runtime needed before DB reset:
   - `docker compose -f docker-compose.yml -f docker-compose.production.yml up -d solfacil-db`
4. Run canonical DB rebuild + production-only seed:
   - `cd backend && ADMIN_PASSWORD_HASH='<bcrypt-hash>' bash ./scripts/bootstrap-production.sh`
   - expected result: `ORG_ENERGIA_001`, `USER_ADMIN_001`, 3 Brazil gateways, and 9 prebuilt minimal assets only
5. Recreate Solfacil app containers only:
   - `docker compose -f docker-compose.yml -f docker-compose.production.yml up -d --force-recreate solfacil-bff solfacil-m1`
6. Verify public acceptance:
   - login cookie contract
   - `/api/auth/session`
   - security headers
   - no browser JWT in `localStorage`
   - only intended production gateways visible

## Stop lines
- Stop if any command wants to restart or edit `nginx-web` unexpectedly.
- Stop if Redis binding attempts to claim host port `6379`.
- Stop if DB reset path still produces demo org/gateway rows.
- Stop if production verifier fails.
- Stop if acceptance shows old auth contract still live.

## Notes
- `docker-compose.production.yml` reuses the already-running host Redis via `host.docker.internal`, avoiding a new 6379 bind on the shared host.
- `bootstrap-production.sh` uses canonical schema+migrations first, then applies a production-only seed layer. This is the correct v6.9 reset path.
