Solfacil v6.10 Runtime Governance Skeleton

這份 DESIGN 把 REQ 收斂成可落地的架構：spine 放在 M9 shared layer，用統一 event contract、lifecycle、health summary、self-check read model 與 /api/runtime/* backend surface，去建立 phase-1 的 runtime governance 骨架。

Status: reviewed / revised Audience: internal ops first Coverage: narrow but deep Surface priority: backend first Source: docs/v6.10-runtime-governance-design.md

你先看這裡

設計一句話： 把 runtime governance spine 安裝進 M9 shared layer，讓各模塊 emit 標準化 facts，由 spine 統一做 event schema、severity、lifecycle、issue projection、self-check 與 health summary。
  • 不是新 monolith，也不是先做一頁漂亮 dashboard。
  • phase-1 只接最關鍵的 runtime path：BFF、DB、M1 ingest、M3 dispatch、scheduler/cron。
  • operator-facing 核心是 /api/runtime/*；UI 只做最小可用的 #runtime。
  • P5/P6 domain surfaces 明確分離，不混寫、不共表、不偷語義。

架構落點

  • backend/src/shared/runtime/* 成為 spine 主體
  • types/runtime.ts 定義 canonical event / issue / lifecycle types
  • 002_runtime_governance.sql 建 runtime tables
  • BFF 暴露 read/write operator endpoints

資料模型決策

  • runtime_events：append-only raw history
  • runtime_issues：單 row projection，reopen 不創第二個 issue id，而是同 row 開新 cycle
  • runtime_self_checks：latest-state read model
  • runtime_health_snapshots：5 分鐘 cadence 的 posture snapshots

Phase-1 onboarding set

SourceWhy firstWhat becomes governed
BFF request lifecycle所有 user-facing path 的入口unhandled exception boundary、auth failure bursts、wrapHandler catch-all
Shared DB pool幾乎所有 runtime path 的共用底座reachability、query timeout、connection failure、self-check results
M1 ingest pipeline沒有新鮮資料就沒有平台真相telemetry stale、fragment backlog、parser failure bursts
M3 dispatch loops最典型的「系統活著但工作沒在做」dispatcher / timeout checker heartbeat、ack stall
Scheduler / cron時間邊界正確性的來源schedule generator、aggregators、billing job heartbeats / errors

Ryan review 後修正的關鍵點

  • 修掉 runtime_issues reopen identity 矛盾：現在明確採用單 row / fingerprint + cycle_count
  • 補齊 runtime_self_checks table，讓 /api/runtime/self-checks 有明確讀模型
  • 澄清 overall: "disabled" 只是 API-level posture，不是 severity/state enum
  • 澄清 redis.* 在 phase-1 只是 reserved namespace，不進固定 component summary

明確 defer 的東西

  • SSE push for runtime issues
  • alert routing（email / Slack / webhook）
  • customer-facing subset
  • trend charts / broader UX polish
  • external-source event ingest
