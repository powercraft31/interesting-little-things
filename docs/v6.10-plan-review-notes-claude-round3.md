## 2. What is now fixed

### 2.1 Gates are now tied to mandatory pass evidence
Section 3 is materially stronger.

Each gate now names explicit required pass evidence rather than only artifact presence:
- Gate A binds bootstrap to WS0 tests plus disabled-default smoke proof.
- Gate B binds storage to first-apply, second-apply, and no-regression DB verification.
- Gate C binds M9 readiness to named WS2 slice suites and explicit lifecycle/failure proofs.
- Gate D binds operator contract readiness to route/auth/envelope/wiring tests plus admin/non-admin smoke.
- Gate E binds emitter onboarding to slice-suite and isolated-integration proof per onboarded slice.
- Gate F and Gate G now also require named evidence sets.

That closes the main “code exists vs behavior proven” gap from round 2.

### 2.2 Flag model now matches the rollout story
WS0 now defines the missing toggle model explicitly:
- one global gate: `runtime_governance_enabled`
- per-slice flags for `bff_db`, `m1_ingest`, `m2_scheduler`, `m3_dispatch`, `m4_billing`, and frontend page exposure

Rollout and rollback sections now use the same flag model. The plan no longer claims incremental enablement without defining the toggles needed to perform it.

### 2.3 Integration verification is now segmented to match enablement order
The former combined integration block has been split into isolated integration sets aligned with rollout:
- I2 substrate
- I3 M1 ingest
- I4a M3 dispatch
- I4b M2 scheduler
- I4c M4 billing
- I5 operator actions
- I6 retention/auto-close
- I7 feature-flag isolation

This is now operationally credible for incremental onboarding. The implementation team has a clear prove-one-slice-before-next path.

### 2.4 Retention / auto-close is now execution-concrete enough
WS10 now pins a phase-1 enforcement path and proof path:
- mandatory shared-layer scheduled retention job
- explicit responsibilities for 90d raw-event retention, 24h recovered auto-close, 72h stale auto-close, and 30d snapshot retention
- local/test execution path via injected clock or overridable cutoffs
- named slice tests and integration set I6 to prove the behavior

That is sufficiently concrete for implementation and acceptance.

### 2.5 Frontend verification lane is now pinned down
WS9 now names a required automated verification target:
- `frontend-v2/test/runtime-page-smoke.test.js`

That fixes the prior “choose the test home yourself” ambiguity. The frontend lane is now sharp enough for handoff.
