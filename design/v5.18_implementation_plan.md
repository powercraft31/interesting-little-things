## PR 6: ScheduleTranslator — bidirectional translation + validation

**Dependencies**: PR 5 (CommandTracker uses it), PR 4 (BMS limits from telemetry for validation)

**Files to modify/create:**

| File | Action | Est. Lines |
|------|--------|-----------|
| `backend/src/iot-hub/translators/schedule-translator.ts` | CREATE | ~200 |
| `backend/src/shared/types/schedule.ts` | CREATE | ~40 |

**Changes:**

**schedule-translator.ts:**
- `parseGetReply(payload)`: protocol → DomainSchedule
  - tariff+charge → peak_valley_arbitrage/charge
  - tariff+discharge → peak_valley_arbitrage/discharge
  - self_consumption → self_consumption
  - peak_shaving → peak_shaving
  - export_policy → allowExport boolean
- `buildConfigSet(clientId, schedule)`: DomainSchedule → protocol message
  - Reverse of parseGetReply
  - All numbers → strings
  - Full SolfacilMessage envelope
- `validateSchedule(schedule)`: throws ScheduleValidationError
  - SOC range: 0-100, min < max
  - Current limits: ≥0, ≤ BMS reported max
  - grid_import_limit: ≥0
  - Slot times: multiples of 60, start < end
  - Slot coverage: union = [0, 1440), no overlap

**schedule.ts:**
- DomainSchedule interface
- DomainSlot interface
- ScheduleValidationError class

**Publish functions (added to connection manager or separate module):**
- `publishConfigGet(clientId)`: build message, log command, publish
- `publishConfigSet(clientId, schedule)`: validate → build → log → publish
