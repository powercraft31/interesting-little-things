## Phase 0: DB Migration（Database Schema v5.10）

### Task 0.1 — 修復 feature_flags UNIQUE 約束

| 項目 | 說明 |
|------|------|
| **目標文件** | `db/migrations/005_v5.10_rls_admin_bypass.sql`（新建） |
| **修改邏輯** | 執行 `ALTER TABLE feature_flags DROP CONSTRAINT IF EXISTS feature_flags_flag_name_coalesce_key;` 然後 `CREATE UNIQUE INDEX IF NOT EXISTS uq_feature_flags_name_org ON feature_flags (flag_name, COALESCE(org_id, ''));` |
| **所屬模組** | Database Schema v5.10 |
| **驗證** | `\d feature_flags` 確認 UNIQUE INDEX 存在；INSERT 重複 flag_name+org_id 應報錯 |

### Task 0.2 — 新增 RLS Admin Bypass 策略

| 項目 | 說明 |
|------|------|
| **目標文件** | `db/migrations/005_v5.10_rls_admin_bypass.sql`（同上，追加） |
| **修改邏輯** | 為以下 10 張表新增 `_admin_bypass` RLS policy：`assets`, `trades`, `revenue_daily`, `dispatch_records`, `tariff_schedules`, `vpp_strategies`, `parser_rules`, `feature_flags`, `trade_schedules`, `algorithm_metrics`。每個 policy 的 USING 條件為 `current_setting('app.current_org_id', true) IS NULL OR current_setting('app.current_org_id', true) = ''` |
| **所屬模組** | Database Schema v5.10 |
| **驗證** | 不設定 `app.current_org_id` 時，`SELECT * FROM assets` 應返回全量數據 |

### Task 0.3 — dispatch_commands 啟用 RLS

| 項目 | 說明 |
|------|------|
| **目標文件** | `db/migrations/005_v5.10_rls_admin_bypass.sql`（同上，追加） |
| **修改邏輯** | `ALTER TABLE dispatch_commands ENABLE ROW LEVEL SECURITY;` + 建立 `rls_dispatch_commands_tenant` 和 `rls_dispatch_commands_admin_bypass` 兩個 policy |
| **所屬模組** | Database Schema v5.10 |
| **驗證** | 設定 `app.current_org_id = 'ORG_ENERGIA_001'` 後，只能查到該 org 的 dispatch_commands |

### 🧪 Test Checkpoint 0

```bash
# 執行 migration
psql -d solfacil_vpp -f db/migrations/005_v5.10_rls_admin_bypass.sql

# 驗證 feature_flags UNIQUE INDEX
psql -d solfacil_vpp -c "\di uq_feature_flags_name_org"

# 驗證 RLS admin bypass（不設 org_id，應返回全量）
psql -d solfacil_vpp -c "SELECT COUNT(*) FROM assets;"

# 驗證 RLS 租戶隔離（設 org_id，應只返回該 org）
psql -d solfacil_vpp -c "SET app.current_org_id = 'ORG_ENERGIA_001'; SELECT COUNT(*) FROM assets;"

# 驗證 dispatch_commands RLS
psql -d solfacil_vpp -c "\d dispatch_commands" | grep "Row Level Security"
```

## 風險與緩解

| 風險 | 影響 | 緩解措施 |
|------|------|---------|
| Phase 1-2 之間 TypeScript 編譯斷裂 | CI pipeline 會失敗 | Phase 1 和 Phase 2 必須在同一 commit 內完成 |
| dispatch_commands 表結構與 DDL 不匹配 | RLS policy 建立失敗 | Task 0.3 先執行 `\d dispatch_commands` 確認表結構 |
| seed 腳本引用其他不存在欄位 | seed 執行報錯 | Task 4.1 全量比對 seed INSERT 與 DDL 欄位列表 |
| BFF test mock 未更新 | de-hardcoding 後測試仍 FAIL | Task 3.1 同步更新 get-dashboard.test.ts 的 mock 設定 |
