## 1. 背景與動機

### 1.1 現狀

系統目前**零認證**：
- 前端 `data-source.js` 硬編碼 demo auth header（`userId: "demo-user", orgId: "ORG_ENERGIA_001", role: "SOLFACIL_ADMIN"`）
- `local-server.ts` 有 demo middleware：沒帶 Authorization header 就自動注入上述硬編碼值
- BFF middleware（`tenant-context.ts`）接受兩種格式：Raw JSON 或 JWT-style（Base64 decode payload，**不驗簽名**）
- 任何人打開 `https://solfacil.alwayscontrol.net/frontend-v2/` 直接進入，無需登入

### 1.2 既有地基

這些**已經存在**，不需要新建：
- `users` 表（`ddl_base.sql`）：`user_id, email, name, hashed_password, is_active, created_at, updated_at`
- `user_org_roles` 表：`user_id, org_id, role, created_at`（junction table，支持一人多組織）
- `organizations` 表 + RLS 雙池架構（v5.11）
- `shared/types/auth.ts`：`Role` enum（4 種）+ `TenantContext` interface
- `shared/middleware/tenant-context.ts`：`verifyTenantToken()`（已支持 JWT Base64 decode）+ `requireRole()`
- `bff/middleware/auth.ts`：`extractTenantContext()` + `apiError()`
- GRANT：`solfacil_app` 和 `solfacil_service` 對所有表都有 SELECT/INSERT/UPDATE/DELETE

### 1.3 設計文件歷史

- `06_IDENTITY_MODULE_v5.2.md`：完整企業級設計（Cognito + SSO + MFA + SAML），**未實作**
- 本次 v5.23 是 **Phase 1：最小可用本地 JWT 認證殼**，不引入 Cognito

## 2. 需求（Alan 2026-03-12 確認）

### 2.1 核心需求

| # | 需求 | 說明 |
|---|------|------|
| R1 | **登入** | 用 email + password 登入，拿到 JWT |
| R2 | **JWT 認證** | 所有 `/api/*` 請求必須帶 JWT，middleware 驗簽名 |
| R3 | **帳號管理** | 不做註冊流程。管理員直接「添加帳號」（同 gateway auto-discover 思路） |
| R4 | **登入頁面** | 前端加 login.html，未登入自動跳轉 |
| R5 | **登出** | 導航列加登出按鈕，清 token 跳回登入頁 |
| R6 | **移除 /frontend-v2/ 路徑** | 只保留 `/` 入口，移除 `/frontend-v2/` 的 nginx location + Express static 掛載 |

### 2.2 不做（Phase 1 範圍外）

| 不做 | 理由 |
|------|------|
| 忘記密碼 / 重設密碼 | Phase 2 |
| 自助註冊 | 不需要，管理員加帳號 |
| MFA / SSO | Phase 2+ (Cognito) |
| 前端頁面級權限檢查 | 靠 RLS 隔離 org_id 就夠 |
| Refresh Token | Phase 1 用單 token，過期重登 |
| user_org_roles 多組織 | Phase 1 每人只屬一個 org（簡化） |

### 2.3 關鍵約束

| 約束 | 詳情 |
|------|------|
| **外層套殼** | 所有現有 handler、cron job、前端六頁面**零改動** |
| **JWT payload 格式** | `{ userId, orgId, role }` — 跟現有硬編碼 JSON header 完全一致 |
| **Auth middleware 輸出** | 驗完 JWT 後覆寫 `req.headers.authorization = JSON.stringify({ userId, orgId, role })`，下游 handler 零感知 |
| **users 表** | 既有 schema 不改結構。`hashed_password` 欄位名沿用（不改為 password_hash） |
| **user_org_roles** | Phase 1 保留但不主動讀寫。login 時 JOIN 取 org_id + role |
| **bcrypt** | 密碼 hash 用 bcryptjs（12 rounds） |
| **jsonwebtoken** | JWT 簽發 + 驗證 |

## 4. 決策點

### 4.1 Login endpoint 用 service pool（已確認）

**理由**：登入時未知 orgId，無法走 app pool (RLS)。僅 SELECT users + user_org_roles WHERE email。
**風險**：低。只讀操作，不洩漏跨 org 數據。

### 4.2 JWT Secret

Phase 1 用環境變數 `JWT_SECRET`，local-server.ts 用固定字串（如 `solfacil-dev-secret`）。
EC2 部署時設正式值。

### 4.3 Token 過期時間

Phase 1：**24 小時**。過期 → 前端 401 → 跳 login 頁。不做 refresh token。
